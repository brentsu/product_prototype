📊 成本计算说明
  • 成本维度：库存成本按"工厂+SKU"维度计算，每个工厂的每个SKU维护一个独立的加权平均成本
  • 加权平均成本：该SKU在指定工厂下的所有入库记录按加权平均法计算出的单位成本
  • 库存成本金额：= 库存数量 × 加权平均成本（库存数量 = 入库数量 - 消耗）
  • 累计出库成本：= 累计出库数量 × 加权平均成本（每次出库时按当前加权成本计算）
  • 成本更新时机：每次采购入库后重新计算加权平均成本，出库时使用当前加权成本计价

📖 加权平均算法说明

算法原理

加权平均法是一种常用的库存成本核算方法。系统按照"工厂+SKU"维度维护独立的加权平均成本，即每个工厂的每个SKU都有自己独立的成本池。每次该工厂+SKU组合入库后重新计算平均单位成本，出库时按当前加权平均成本计价。

📐 成本计算维度

成本池 = 工厂 + SKU
例如：460工厂 + LC001-1-L 是一个独立的成本池
同一SKU在不同工厂下维护不同的加权平均成本

📐 计算公式

入库时：
新加权成本 = (原库存金额 + 本次入库金额) ÷ (原库存数量 + 本次入库数量)

出库时：
出库成本 = 出库数量 × 当前加权成本
剩余库存金额 = 原库存金额 - 出库成本
加权成本保持不变

💡 计算示例（460工厂 + LC001-1-L）

成本池：460工厂 + LC001-1-L

初始状态：库存0件，加权成本¥0.00

第1次入库：460工厂的LC001-1-L入库10,000件，单价¥20.00

→ 加权成本 = (0 + 10,000×20) ÷ (0 + 10,000) = ¥20.00

→ 库存：10,000件，库存金额¥200,000

→ 460工厂+LC001-1-L的加权成本：¥20.00

第1次出库：460工厂的LC001-1-L出库6,000件

→ 出库成本 = 6,000 × ¥20.00 = ¥120,000（按当前加权成本计算）

→ 剩余库存：4,000件，库存金额¥80,000

→ 加权成本保持：¥20.00

第2次入库：460工厂的LC001-1-L再次入库5,000件，单价¥22.00

→ 加权成本 = (80,000 + 5,000×22) ÷ (4,000 + 5,000) = ¥20.89

→ 库存：9,000件，库存金额¥188,000

→ 460工厂+LC001-1-L的新加权成本：¥20.89

说明：如果同一SKU（LC001-1-L）从其他工厂（如470工厂）入库，将维护独立的成本池和加权成本，互不影响。

✅ 优势特点
  • 按工厂+SKU维度独立核算：每个工厂的每个SKU维护独立的成本池，反映不同工厂的实际采购成本差异
  • 计算简便，易于理解和操作：加权平均法逻辑清晰，易于财务人员理解和使用
  • 避免物价波动对成本的剧烈影响：平滑成本波动，避免单次采购价格波动对整体成本造成冲击
  • 成本计算相对平稳：符合会计稳健性原则，便于成本分析和预测
  • 适合批量采购、价格相对稳定的商品：尤其适合跨境电商批量采购场景
  • 便于财务核算和税务处理：成本追溯清晰，满足财务合规和审计要求
  • 支持多工厂成本对比：同一SKU在不同工厂下的成本可独立对比分析

📖 字段说明

入库数量

从工厂采购入库的总数量，此字段一经录入不可更改，保持历史记录的完整性

消耗

因销售出库而消耗的库存数量，用于计算可用库存和财务成本结转

结算数

与工厂约定的结算数量，通常在一段时间后确认，用于财务应付账款核算

退货数

因质量问题等原因退回给工厂的数量，冲减应付账款

真实结算数

实际需要与工厂结算的数量，计算公式：真实结算数 = 结算数 - 退货数

💡 财务合规关键要点

  • 数据不可篡改："入库数量"字段一经录入不可修改，确保历史数据的完整性和可追溯性
  • 退货处理：退货不减少"入库数量"，而是通过"退货数"字段单独记录，保持审计线索清晰
  • ⚠️ 退货限制：只有在工厂未结算前才允许退货，一旦工厂结算完成（结算数>0），则不允许再进行退货操作
  • 财务结算：最终与工厂结算时，使用"真实结算数"（结算数-退货数）作为计算依据
  • 库存消耗："消耗"字段仅记录销售出库，不包含退货，两者分别核算
  • 不能超量消耗：可用库存=入库数量-消耗-退货数，出库数量不能超过可用库存，系统会自动拒绝超量消耗请求
  • 全程可追溯：所有变更都在"库存池变更记录"中留痕，支持完整的审计追溯
